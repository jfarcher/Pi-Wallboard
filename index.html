Pi-Wallboard

Pi wallboard is essentially a digital display solution, configuring a stock Raspbian install to launch Chromium in kiosk mode on boot. Chromium will display a given URL from a stored configuration file, restarting the browser should the configuration file be modified.

Using the tvservice tools supplied with Raspbian cron jobs are also added to put the attached display into standby mode, resulting in the display going to sleep should it support this feature.

This wallboard solution is ideal for datacentre monitoring and information display. eg. call centres, and IT operations centres.

Installation

Installation is very straightforward,

git clone https://github.com/jfarcher/Pi-Wallboard

cd Pi-Wallboard

chmod +x config.sh

sudo ./config.sh

Complete the install by answering any questions presented.

A puppet module is also included to allow central configuration, ideal if multiple screens are to be deployed. Simply copy the puppet directory to your puppetmasters modules directory

cp -r puppet /etc/puppet/modules/wallboards

the files directory contains a sample configuration, these should be stored in the format wallboard.conf-hostname. Replacing hostname with the actual short hostname of your raspberry pi.

Should you not employ a puppet environment this will be recognised during the install and relevant puppet based configuration be applied.
